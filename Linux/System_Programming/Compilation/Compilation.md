## 预编译

### 头文件展开

预编译中会对头文件进行展开，所有包含的类似于：`#include <...>` 或 `#include ""` 都会被展开，即直接把头文件的内容拷贝到包含头文件的源文件中。在 gcc 编译器中（本文仅以 GCC 举例，G++ 和 GCC 的操作高度相似，直接套用即可），添加 `-E` 选项可以让文件在完成预编译阶段后停下来。

```linux
gcc -E code.c -o code.i
```

`-o` 选项是生成指定的目标文件，对预编译后的文件后缀命名为 `*.i`。

![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p1.png)

打开 code.i 后：

![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p3.png)

前面总共有八百多行，即对头文件 `stdio.h` 进行了展开。（预编译讲解皆参考此图和 code.c 的内容对比）



### 条件编译

在 code.c 中所写的：

```cpp
#ifdef ILOVEYOU
     printf("I LOVE YOU\n");
#else
     printf("YOU DONT LOVE ME\n");
#endif
```

由于 “ILOVEYOU” 未被定义，故应当执行 else 的部分，可见在 code.i 中直接变成了：`printf("YOU DONT LOVE ME\n");` 这一过程就是完成条件编译。
[^条件编译的应用]



### 宏替换

在 code.c 中，我们所打印的是 NUM 这个宏，而在 code.i 中直接变成了 1024，可以看到宏仅仅是完成了简单的替换任务，所以存在不会有类型检查，在预处理阶段就展开，无法调试等诸多问题。



### 删除注释

注释仅是给编写源代码的人提供提示和帮助，在程序执行时，注释对编译器而言没有任何意义，故所有的注释都会在这一阶段删除。

## 编译

编译过程会将预编译后的程序转换为汇编语言。使用 `-S` 选项，让文件完成编译后停下来。此时既可以对 `*.i` 文件进行 `-S` 选项后停止在编译完成后阶段，也可直接对源文件编译，效果一致，对于完成编译后的文件，后缀一般为：`*.s`。

```linux
gcc -S code.i -o code.s
```



![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p4.png)

此时用 vim 打开 code.s：

![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p5.png)

会发现所对应的基本都为汇编指令。

## 汇编

汇编是将所对应的汇编文件（code.s）翻译为机器码，即二进制代码的过程。添加 `-c` 选项即可。

```linux
gcc -c code.s -o code.o
```

所对应的二进制文件，一般命名为：`*.o`，即 object 的意思。

## 链接

默认不添加任何选项即可完成链接过程后停止，生成可执行程序。

```linux
gcc code.o -o code
```

![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p10.png)

### 什么是链接

例如在程序中调用了 `printf` 函数，但这个函数并不是我们实现的，而是包含在 `stdio.h` 中，在库中实现。故链接就是通过编译器获得库的位置，从而去调用库中函数的内容以实现功能的方式。

### 什么是库

上述对链接的描述和函数调用类似，但其和函数不同的是它可以实现仅提供接口不提供实现方法（开源库除外），实现方法都在库内部，故 `printf` 函数又叫库函数，它们一般被编译为二进制并打包，编译在一般情况下是不可逆的，故无法直接查看库函数的实现。

对于库函数：

1. 提高开发效率：库函数提供了常见功能的实现，程序开发者可以直接调用这些函数，避免从头开始编写重复的代码。
2. 减少错误和调试难度：库函数经过测试和优化，通常比自己编写的代码更可靠。使用库函数可以减少出错的机会，且如果出现问题，也更容易进行调试。
3. 代码复用和维护：将常用功能封装为库函数，可以在多个项目或模块中复用代码，提高代码的可维护性和可重用性。

### 动态库

动态库是在程序运行时被链接到目标程序的库。例如有一天你做完了工作后想去喝咖啡，于是你去了咖啡厅并买了咖啡，完成了这一过程。

![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p8.png)

其它步骤在家里都可以完成，但是没办法在家里喝咖啡，所以必须要去找咖啡厅（动态库）。而想喝咖啡的人肯定不止一个，只要是想喝咖啡，且无法在家完成这一过程的时候，必然会来到咖啡厅，所以咖啡厅可以容纳很多人（动态库可以被很多程序调用）。但如果有一天咖啡厅因为经营不善倒闭了，那么所有任务清单上有喝咖啡这一项任务的，且无法在家里完成这一过程的人发现咖啡厅倒闭了，没办法完成这一任务（程序报错，找不到对应的动态库）。



### 静态库

静态库是在编译时被链接到目标程序中的库。就类似于在家里买了一台咖啡机，随时随地都可以喝咖啡。

![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p9.png)

于是不需要再去咖啡厅（不用调用动态库），而是自己在家里完成这一过程（调用静态库）。这个时候，咖啡厅是否倒闭与你无关，在家里咖啡机即可满足喝咖啡的需求。但咖啡机需要占用家里的一些地方，咖啡厅却不用（静态链接的大小比动态链接要大，因为要把库在连接时就直接加载到程序中）。



### 优劣分析

静态库优点：

1. 程序可以做到独立，不依赖外部库文件。

静态库缺点：

1. 体积很大，消耗资源多。

动态库优点：

1. 代码量小，可以节省磁盘空间，内存空间，网络空间（如果涉及到网络传输）。
2. 具有共享性，可被多个程序调用，避免了代码冗余。

动态库缺点：

1. 缺失后会影响多个程序，牵一发而动全身。



### 链接方式

![](https://thepale-2022-1313448760.cos.ap-beijing.myqcloud.com/Compilation-p11.png)

通过编译时添加 `-static` 选项来改为链接静态库。可以看到静态库链接比动态库链接大了一百倍左右。同时也可以用 file 命令来查看可执行程序采用何种链接方式。

在编译时默认链接动态库，如果没有动态库则会链接静态库；若加上 `-static` 选项则改变链接优先级，程序会先链接且只能链接静态库，如果链接不到静态库则会报错。

## 补充说明

* 在编译过程中，两个编译的意义不一致，一个是指总共四个程式的大过程，统称为编译；一个是指这四个程式中的一个过程，仅编译过程。
* 程序在编译时默认为 release 版本，加上 `-g` 选项可以让程序变为 debug 版本，会给程序添加些许调试信息，debug 的程序大小也会略大于 release。
* 程序可以同时链接动态库和静态库，不一定只能链接一种，但加上 `-static` 会把所有的链接都改为静态链接。

[^条件编译的应用]:如果需要对一个软件的不同使用版本（例如 VS2022 的 Community 和 Enterprise），它们往往有着部分一样的功能，但付费的版本拥有更多的功能。这一实现不可能是通过同时做两个应用程序来解决的，但可以通过条件编译来实现，检索用户信息，然后决定功能的开放与否。（仅举例使用，并非代表真正的实现逻辑）
